# 研发新人上手 FAQ（演示知识库）

> 本文档为演示用虚构文档，与运维手册、接口文档共同构成检索评测语料。

## 1. 账号与权限

- 域账号由 HR 入职当天开通，默认包含邮箱与 OA
- Git 仓库权限在「权限中心」申请，选择「研发-代码仓库-开发者」角色，审批人为直属组长
- 生产环境堡垒机权限默认不开放，确需排障时申请临时权限，有效期 24 小时
- VPN 使用 EasyConnect，地址为 vpn.example.com，首次登录需绑定手机令牌

## 2. 开发环境

- 统一使用公司镜像源：Maven 配置 nexus.example.com，npm 配置 npm.example.com
- 本地 JDK 建议用 SDKMAN 管理，项目根目录的 `.sdkmanrc` 声明了版本
- 前端 Node 版本以各仓库 `.nvmrc` 为准，使用 fnm 或 nvm 切换
- 数据库开发库连接串在配置中心「dev 环境」查看，禁止连接生产库做开发调试

## 3. 代码规范

- 分支模型：`feature/工号-简述` 开发，合入 `develop` 需要至少 1 名 reviewer
- 提交信息格式：`type(scope): subject`，type 取 feat/fix/docs/refactor/chore
- 合并方式统一 squash merge，禁止直接 push `master` 与 `develop`
- 代码扫描不通过（SonarQube 质量门禁）会自动阻断合并

## 4. 构建与发布

- CI 平台为 Jenkins，流水线配置在各仓库 `Jenkinsfile`
- 测试环境每天 10:00 与 16:00 两个固定发布窗口，紧急修复走值班审批
- 制品统一推送到 Harbor 仓库，镜像 tag 规则为 `分支名-短commit`
- 预发环境发布需要测试负责人在发布单上签字确认

## 5. 常见问题

### 5.1 Maven 依赖下载失败

检查 `settings.xml` 是否指向 nexus.example.com；内网访问外网仓库会被防火墙拦截，报 Connection timed out 属正常现象，切镜像源即可。

### 5.2 本地起服务连不上开发库

开发库仅对办公网段开放，居家办公需先连 VPN；连上仍失败时确认账号是否有对应 schema 的授权。

### 5.3 消息队列本地调试

本地不建议连接共享开发 MQ，容易把队列消息消费掉影响他人；用 docker 起本地 RabbitMQ，或申请个人专属 vhost。

### 5.4 找不到接口文档

各系统接口文档统一发布在知识库平台「接口文档」空间，资源中心相关接口以《资源中心开放接口文档》为准。

## 6. 值班与响应

- 研发值班一周一轮，值班表在 OA「值班管理」查看
- 值班期间收到告警先在群里认领，再按运维手册的故障分级流程处理
- 夜间告警 15 分钟未认领会自动升级到组长
